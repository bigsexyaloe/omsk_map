# Техническое задание (ТЗ)

## Проект: Интерактивная карта: Заводы Омска времён Великой Отечественной войны

---

## 1. Цель и назначение

Разработать веб-сайт с интерактивной картой (Leaflet + OpenStreetMap), на которой размещены метки ключевых промышленных предприятий Омска, эвакуированных в город в годы Великой Отечественной войны (1941–1945). При клике на метку справа от карты отображается окно с фотографией и подробным описанием завода.

Сайт предназначен для жителей Омска, интересующихся историей города, а также для туристов и образовательных учреждений.

---

## 2. Целевая аудитория

- **Количество пользователей:** 50–100 уникальных посетителей в день.
- **География:** преимущественно Россия (г. Омск и туристы).
- **Устройства:** десктопы и планшеты (мобильная версия не является приоритетом на старте).

---

## 3. Макет страницы

Страница состоит из трёх вертикальных блоков:

```
┌──────────────────────────────────────────────────────┐
│                   Шапка (header)                      │
├──────────────────────────────────┬───────────────────┤
│                                  │                    │
│       Карта (Leaflet + OSM)      │   Сайдбар          │
│       65% ширины                 │   35% ширины       │
│                                  │                    │
├──────────────────────────────────┴───────────────────┤
│                   Подвал (footer)                     │
└──────────────────────────────────────────────────────┘
```

- **Шапка:** тёмная плашка с названием проекта и логотипом, фиксированной высоты.
- **Основной блок:** flex-контейнер, занимающий всё свободное пространство между шапкой и подвалом. Содержит карту (слева) и сайдбар (справа).
- **Подвал:** тёмная плашка с копирайтом и примечанием об источниках данных, фиксированной высоты.

---

## 4. Функциональные требования

### 4.1. Основной экран — карта

- На странице отображается интерактивная карта (Leaflet + OpenStreetMap), занимающая левую часть экрана (~65% ширины) между шапкой и подвалом.
- На карте установлены кастомные метки (пины) в местах расположения заводов.
- Карта масштабируется и перемещается стандартными средствами Leaflet.
- Начальный центр карты — центр Омска, начальный зум вычисляется автоматически по границам всех меток.
- **Действия пользователя ограничены:** установлены `maxBounds` (границы по крайним точкам заводов с отступом ~7 км), зум ограничен 10–16 уровнями. Пользователь не может увести карту за пределы Омска.

### 4.2. Панель информации (сайдбар)

- Справа от карты расположена панель (сайдбар), занимающая ~30–40% ширины экрана.
- При первой загрузке страницы в сайдбаре отображается приветственное сообщение: «Заводы Омска в годы Великой Отечественной войны. Нажмите на метку на карте, чтобы узнать подробнее».
- При клике на метку на карте в сайдбаре отображается:
  - Фотография завода.
  - Название предприятия.
  - Полное описание (2–5 предложений).
- При клике на другую метку содержимое сайдбара плавно заменяется на информацию о новом месте.
- Сайдбар имеет вертикальный скролл при длинном контенте.

### 4.3. Метки на карте

- Каждая метка:
  - Имеет уникальный ID.
  - Привязана к координатам (latitude, longitude).
  - Содержит название завода (всплывающий балун при наведении — опционально).
- При клике на метку вызывается событие загрузки контента в сайдбар.
- Цвет/стиль метки — кастомный, единый для всех точек.

### 4.4. Данные

- Все данные о заводах хранятся в JSON-файле на сервере.
- Структура записи:
  ```json
  {
    "id": 1,
    "title": "Завод № 173 (Омский завод транспортного машиностроения, «Трансмаш»)",
    "description": "Создан на базе эвакуированных Ворошиловградского, Конотопского, Брянского и Ленинградского заводов. Стал заводом танковой промышленности № 174, выпускавшим танки.",
    "coordinates": [54.937277, 73.288909],
    "image": "zavod-173.jpg"
  }
  ```
- Изображения хранятся в директории `frontend/assets/images/`.
- Количество точек — 6 (на первом этапе, может быть расширено).
- Данные статичны, обновляются вручную через правку JSON-файла и добавление изображений.

## 5. Нефункциональные требования

### 5.1. Производительность

- Время загрузки страницы — не более 3 секунд при стандартном соединении.
- Размер передаваемых данных минимален: JSON с 6 точками (~3 КБ), статические HTML/CSS/JS.
- Backend лёгкий — FastAPI, без подключения к БД, только чтение JSON-файла.

### 5.2. Надёжность

- При недоступности backend (JSON-данных) карта всё равно отображается (данные кэшируются или хардкодятся на фронтенде как fallback).
- Backend stateless — легко перезапускается.

### 5.3. Безопасность

- Сайт не требует аутентификации и авторизации.
- Картографический провайдер OpenStreetMap не требует API-ключа.
- CORS настроен для запросов с Nginx на backend.

### 5.4. Технологический стек

- **Frontend:** HTML5, CSS3, Vanilla JavaScript (без фреймворков, для минимального размера и простоты).
- **Карты:** Leaflet + OpenStreetMap (бесплатно, без API-ключа).
- **Backend:** Python 3.11+ / FastAPI.
- **Хранение данных:** JSON-файл.
- **Веб-сервер:** Nginx (раздача статики + reverse proxy на FastAPI).
- **Контейнеризация:** Docker + Docker Compose.

### 5.5. Совместимость

- Браузеры: последние 2 версии Chrome, Firefox, Edge, Yandex Browser.
- ОС: Windows, macOS, Linux.

---

## 6. Ограничения и допущения

- Backend выполняет только одну функцию — отдаёт JSON со списком мест по GET `/api/places`.
- База данных не требуется — данные статичны и малы (6 точек).
- Система управления контентом (CMS) не разрабатывается.
- Мониторинг и логирование не предусмотрены.
- Административная панель не требуется.
- Мобильная адаптация не является приоритетом.
- Сайт одностраничный (SPA) без маршрутизации.

## 7. Сценарии использования

### 7.1. Пользователь открывает сайт
1. Пользователь переходит по URL.
2. Nginx отдаёт index.html.
3. Загружается карта Leaflet с тайлами OpenStreetMap.
4. JavaScript инициализирует карту с центром в Омске.
5. Выполняется GET-запрос к `/api/places` за списком точек.
6. На карту наносятся 6 меток заводов.
7. В сайдбаре отображается приветственное сообщение.

### 7.2. Пользователь кликает на метку завода
1. Пользователь кликает на метку на карте.
2. JavaScript определяет ID метки.
3. Находит соответствующий объект в загруженном JSON.
4. Обновляет содержимое сайдбара: фото завода, название, описание.
5. Сайдбар плавно анимирует появление нового контента.

### 7.3. Обновление данных
1. Разработчик редактирует `places.json` (добавляет завод или правит описание).
2. Добавляет/заменяет изображение в `assets/images/`.
3. Перезапускает Docker-контейнер: `docker compose up --build`.
4. Новые данные становятся доступны пользователям.
